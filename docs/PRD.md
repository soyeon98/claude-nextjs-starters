# 노션 기반 견적서 웹 공유 서비스 MVP PRD

> 문서 버전: v2.0 | 최종 수정: 2026-06-04 | 변경: 단일 관리자 구조 전환

## 핵심 정보

**목적**: 노션에서 작성한 견적서를 클라이언트가 별도 가입 없이 웹 URL로 확인하고 PDF로 다운로드할 수 있게 한다.
**사용자**: 노션을 업무 도구로 사용하는 서비스 운영자(관리자 1명) (견적서 발행자), 그리고 견적서를 받는 클라이언트 (수신자)

---

## 사용자 여정

### 견적서 발행자 (사업주) 여정

```
1. 로그인 페이지
   ↓ 이메일/비밀번호 입력 후 로그인 버튼 클릭

2. 견적서 목록 페이지 (대시보드)
   ↓ 노션에서 가져온 견적서 목록 확인
   [미리보기 클릭] → 견적서 미리보기 페이지
   [공유 링크 생성 클릭] → 공유 링크 생성 처리 후 목록 갱신
   [링크 복사 클릭] → 클립보드에 URL 복사 (현재 페이지 유지)

3. 견적서 미리보기 페이지
   ↓ 발행자가 클라이언트에게 보내기 전 확인
   [뒤로 가기] → 견적서 목록 페이지

4. 완료 → 공유 URL을 클라이언트에게 전달
```

### 클라이언트 (견적서 수신자) 여정

```
1. 공개 견적서 뷰 페이지 (고유 URL 접속)
   ↓ 발행자가 공유한 URL로 직접 접속 (로그인 불필요)
   [유효한 URL] → 견적서 내용 표시
   [만료/없는 URL] → 만료 안내 페이지

2. 공개 견적서 뷰 페이지
   ↓ 견적서 내용 확인
   [PDF 다운로드 버튼 클릭] → PDF 파일 다운로드 후 현재 페이지 유지

3. 완료 → 클라이언트가 견적서 내용 확인 및 PDF 보관
```

## 기능 명세

### 1. MVP 핵심 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|---------------|------------|
| **F001** | 노션 API 연동 | `.env`에 설정된 노션 Integration 토큰과 데이터베이스 ID로 견적서 데이터를 가져옴 | 서비스의 데이터 소스 - 이 없이 서비스 불가 | 노션 연동 설정 페이지 |
| **F002** | 견적서 목록 조회 | 연동된 노션 데이터베이스에서 견적서 페이지 목록을 가져와 표시 | 발행자가 어떤 견적서를 공유할지 선택하는 핵심 화면 | 견적서 목록 페이지 |
| **F003** | 공유 링크 생성 | 특정 견적서에 대한 고유 UUID 기반 공개 URL을 생성하고 저장 | 클라이언트 공유의 핵심 기능 | 견적서 목록 페이지 |
| **F004** | 견적서 웹 렌더링 | 노션 페이지 데이터를 파싱하여 브랜드 견적서 형태로 렌더링 | 클라이언트가 보는 핵심 결과물 | 공개 견적서 뷰 페이지 |
| **F005** | PDF 다운로드 | 렌더링된 견적서를 PDF로 변환하여 다운로드 | 클라이언트의 핵심 니즈 - 서류 보관 목적 | 공개 견적서 뷰 페이지 |
| **F006** | 견적서 미리보기 | 발행자가 공유 전 클라이언트에게 보이는 화면과 동일하게 미리 확인 | 발행 오류 방지를 위한 필수 검토 기능 | 견적서 미리보기 페이지 |

### 2. MVP 필수 지원 기능

| ID | 기능명 | 설명 | MVP 필수 이유 | 관련 페이지 |
|----|--------|------|---------------|------------|
| **F010** | 기본 인증 | 이메일/비밀번호 로그인, 로그아웃 | 발행자 데이터(노션 토큰) 보호를 위한 최소 인증 | 로그인 페이지 |
| **F012** | 공유 링크 관리 | 생성된 공유 링크 목록 확인, URL 복사, 링크 비활성화 | 발행한 견적서 공유 현황 관리 | 견적서 목록 페이지 |

### 3. MVP 이후 기능 (제외)

- 견적서 승인/거절 워크플로우 (클라이언트 서명)
- 견적서 조회 통계 (클라이언트 열람 시각, 횟수)
- 이메일 발송 기능 (URL을 직접 이메일로 발송)
- 견적서 만료일 설정
- 다중 노션 데이터베이스 연동
- 커스텀 견적서 템플릿 편집기
- 견적서 버전 관리
- 팀/멤버 관리

| **주요 기능** | - `.env`에 설정된 노션 Integration 토큰/데이터베이스 ID의 연동 상태 표시 (설정됨/미설정)<br>- **연동 테스트** 버튼: 실제 Notion API 호출로 연결 확인<br>- 연동 성공/실패 결과를 인라인 메시지로 표시<br>- 미설정 시 `.env` 설정 안내 문구 표시 |

### 공개 견적서 뷰 페이지

> **구현 기능:** `F004`, `F005` | **인증:** 불필요 (공개 페이지)

| 항목 | 내용 |
|------|------|
| **역할** | 클라이언트가 고유 URL로 접속하여 견적서를 확인하고 PDF로 다운로드하는 핵심 공개 페이지 |

## 기술 스택

### 프론트엔드 프레임워크

- **Next.js 16** (App Router, Turbopack) - 현재 프로젝트 프레임워크
- **TypeScript 5.x** - 타입 안전성
- **React 19** - UI 라이브러리

### 스타일링 & UI

- **Tailwind CSS v4** (설정 파일 없는 새 엔진) - 현재 프로젝트 스타일 시스템
- **shadcn/ui** (radix-nova 스타일) - 현재 프로젝트 컴포넌트 라이브러리
- **Lucide React** - 아이콘

### 폼 & 검증

- **React Hook Form 7.x** - 폼 상태 관리 (현재 프로젝트 사용 중)
- **Zod 4.x** - 스키마 검증 (현재 프로젝트 사용 중)

### 핵심 신규 의존성

| 패키지 | 버전 | 용도 |

| `@notionhq/client` | ^2.x | Notion API 공식 클라이언트 |
| `notion-to-md` | ^3.x | 노션 블록을 마크다운/구조화 데이터로 변환 |

## MVP 범위 및 제외 항목

### MVP 포함

- 이메일/비밀번호 기반 관리자 로그인
- `.env`에 설정된 노션 Integration 토큰 + 데이터베이스 ID 기반 연동
- 노션 데이터베이스 페이지 목록 조회
- 고유 UUID 토큰 기반 공유 URL 생성
- 기본 견적서 템플릿 렌더링 (항목, 금액, 발행자/수신자 정보)
- `@react-pdf/renderer` 기반 클라이언트 사이드 PDF 다운로드
- 공유 링크 비활성화

### MVP 제외

- OAuth (구글, 노션 OAuth 로그인)
- 견적서 열람 통계
- 이메일 발송
- 견적서 만료일 설정
- 커스텀 브랜딩 (로고, 색상 변경)
- 클라이언트 서명/승인 기능
- 다국어 지원
- 견적서 직접 편집 (노션 외부에서)
